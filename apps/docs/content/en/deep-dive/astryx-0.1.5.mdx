## CSS layers and Theme

Declare reset, core base, and Neutral theme order so token precedence is predictable. Tailwind and plain CSS consumers can use the pre-built Neutral theme directly without running the Astryx CLI.

```tsx
import { Theme } from "@astryxdesign/core/theme";
import { neutralTheme } from "@astryxdesign/theme-neutral/built";
import "@astryxdesign/theme-neutral/theme.css";

export function App({ children }: { children: React.ReactNode }) {
  return <Theme theme={neutralTheme}>{children}</Theme>;
}
```

Override public CSS tokens after the theme import when the application needs different fonts, radii, or motion values.

## Compose the frame

`AppShell` owns page height and mobile navigation and receives `TopNav` and `SideNav` slots. Long documents use `height="auto"` for page scrolling. `LinkProvider` connects Astryx navigation to Next Link.

## Interaction components

`MobileNav` and `CommandPalette` use native dialog behavior for focus trapping, Escape, backdrop, and scroll locking. `Outline` targets heading IDs and supplies scroll spy and an active indicator.

## Practical tips

- Pin core and theme-neutral to the same 0.1.5 release.
- Use the Tailwind token bridge for layout and stable component selectors only where nested document content requires them.
- Model theme mode as one `light | dark | system` value.
- Preserve complete meaning and function in Tier 1; Tier 2 may simplify effects but must remain navigable.

## Failure cases

### Overriding internal DOM with CSS

Internal classes and structure are not a stable contract. Prefer public props, tokens, and documented stable selectors.

### Duplicating focus management

Extra global focus and scroll-lock logic can conflict with native dialogs. Control Astryx open state and callbacks instead.
