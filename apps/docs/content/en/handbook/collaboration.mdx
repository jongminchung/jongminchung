## Issues

1. Before submitting an issue, check whether it already exists in the issue tracker.
2. When submitting a bug, provide a minimal reproduction scenario.
3. When submitting an issue, pull request, or merge request, clearly describe the problem being solved or document the design of the feature being added.
4. Pull requests and merge requests should be linked to an issue, although this is not mandatory. If no issue is linked, include enough issue context in the description.
5. Every change must be tested: unit tests, integration tests, and E2E tests when possible.
