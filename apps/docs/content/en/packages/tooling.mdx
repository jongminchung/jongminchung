## Overview

`@jongminchung/tooling` centralizes formatter and linter defaults plus aliases derived from package exports. Consumers declare small overrides instead of copying configuration.

## oxlint

```ts
import { defineOxlintConfig } from "@jongminchung/tooling/oxlint";

export default defineOxlintConfig({
  rules: { "eslint/no-console": "warn" },
});
```

The default enables type-aware analysis and checks unused values, floating promises, and misused promises. Override only rules for which the caller has an intentional exception.

## oxfmt

```ts
import { defineOxfmtConfig } from "@jongminchung/tooling/oxfmt";

export default defineOxfmtConfig({
  printWidth: 100,
  ignorePatterns: ["generated/"],
});
```

Base ignore patterns and overrides remain in place, with caller arrays appended. JSON uses two spaces, and package scripts and imports are sorted.

## package-map

```ts
import { createViteResolveAliases } from "@jongminchung/tooling/package-map";

export default {
  resolve: { alias: createViteResolveAliases({ rootDir: process.cwd() }) },
};
```

The package reads root workspaces and package `exports` to generate TypeScript paths or Vite aliases. It resolves `source`, `default`, `import`, and `types` conditions in order while preserving CSS and JSON subpaths.

## API

- `defineOxlintConfig`, `defineOxfmtConfig`: merge defaults and overrides
- `loadWorkspacePackageMap`: read workspace packages and manifests
- `createPackageExportAliases`: connect public subpaths to real targets
- `createTsconfigPaths`, `createTsconfigAliasConfig`: generate TypeScript paths
- `createViteResolveAliases`: generate regular-expression Vite aliases
- `formatTsconfigAliasConfig`, `writeTsconfigAliasConfig`: serialize and write alias files
