## Status

`@jongminchung/ui` 1.0.0 preserves its API and distribution contract for existing consumers but receives no new features. Use Astryx for new UI. Removing the package or changing a 1.0.0 public export is outside this migration.

## Migrate to Astryx

1. Wrap the app in Astryx `Theme` and `AppShell`.
2. Declare reset, base, and Neutral theme CSS layers.
3. Replace `Button`, `Badge`, and `Card` one surface at a time.
4. Remove Tailwind token dependencies, then verify keyboard, dark mode, and responsive behavior.

```tsx
import { Card } from "@astryxdesign/core/Card";
import { Theme } from "@astryxdesign/core/theme";
import { neutralTheme } from "@astryxdesign/theme-neutral";

export function App() {
  return (
    <Theme theme={neutralTheme} mode="system">
      <Card padding={4}>Migrated content</Card>
    </Theme>
  );
}
```

## 1.0.0 API

### Components

- `Button`, `buttonVariants`, `ButtonProps`
- `Badge`, `badgeVariants`, `BadgeProps`
- `Card`, its six composition components, and their prop types

### Design tokens and utilities

- `designSystem`, `DesignSystem`: color, type, and spacing data from the archived design analysis
- `cn`: class merging with `clsx` and `tailwind-merge`

### CSS subpaths

`styles.css`, `tokens.css`, and `baseline.css` remain available for 1.0.0 consumers. Do not import them into an Astryx app.
